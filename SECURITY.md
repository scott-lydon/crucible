# Security Policy

## Reporting a vulnerability

Crucible is adversarial verification tooling, so we take security reports
seriously. Please report vulnerabilities privately rather than in public issues.

Use GitHub's private vulnerability reporting on this repository: the
"Report a vulnerability" button under the Security tab. We aim to acknowledge
reports within a few days.

## Scope

In scope: the orchestrator, the oracle ensemble, the producer sandbox boundary,
and the sealed-spec resolver. Out of scope: third-party dependencies (report
those upstream) and any deployment you operate yourself.

## Supported versions

Crucible is pre-1.0. Security fixes land on `main`.
