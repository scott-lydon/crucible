This is the run launcher, where an operator configures an evaluation. You pick the target system to attack, the code agent or the fraud model, each registered with its own adapter. The sealed specification is shown as YAML. Sealing hashes the spec and locks it, so the producer and the target never see the file they are being judged against. You set the rounds and the budget, and only a sealed spec enables the run, which then launches a sealed sandbox and opens the live run.
