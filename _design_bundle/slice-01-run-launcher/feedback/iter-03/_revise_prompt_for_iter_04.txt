1. ERROR-STATE LIVE-UPDATES PIP (consensus: bank + codegen). The error frame renders the "Live updates · connected" pip GREY (#3A4654) with a small check, while happy/empty render the identical "connected" label with a GREEN pip (#57C08A + 0 0 0 3px #57C08A22 glow ring). Same transport state signalled two different ways reads as a false "degraded" signal. CHANGE: in the error state, render the pip GREEN identical to happy/empty, because the transport IS still connected; the red PARSE ERROR banner already carries the run-health severity, so the transport pip must not be overloaded to also mean run-health. (This deliberately REVERSES the iter-2 Proc-4 "demote pip to grey-with-check" instruction, which two reviewers now find confusing; if a future state genuinely shows a degraded transport, change the LABEL to "reconnecting"/"degraded" rather than recoloring an unchanged "connected".)

2. COST CHIP CAPTURE TIMESTAMP (bank, audit-posture). The EST/RUN chip carries cost-model provenance ("cost-model v2.1 · 4c7b") but no "as of" capture time, and the SESSION chip ($61.40 / $250.00 mo) carries neither timestamp nor freshness marker on its face; the estimate/ledger drawer header also lacks a capture timestamp. A dollar figure shown to a risk officer without an as-of time cannot be trusted as fresh-vs-stale. CHANGE: add a compact monospace "as of 14:07:52Z" caption (>=12px, satisfying the procurement type-size floor) to the EST/RUN and SESSION chip faces and to the drawer header, using the same clock the acknowledgment caption uses. (This is a deliberate, documented partial reversal of iter-2 CodeGen-1, which moved "as of" into the drawer for density; per the accessibility/audit tiebreaker, audit-posture wins, so the capture time returns to the face but kept to ONE compact line so it does not re-crowd the primary $ scalar.)

3. EMPTY-STATE STEP-COUNT RECONCILIATION (bank). The guidance bar says the three inputs are required and form labels read "of 3", but the disabled-Start caption says "4 steps remaining" (three inputs + judge acknowledgment = four gates), and the copy never reconciles the 3-vs-4 count. CHANGE: itemize the gate count so the integer is never bare. Make the disabled-Start caption read "4 gates remaining: target, spec, budget, judge ack" and align the guidance bar to name the judge acknowledgment as the explicit fourth gate. Keep the form section numbering consistent with whichever count you state.

CHANGE: demote the three RUN SUMMARY placeholders from bordered dashed chips to borderless muted text (em-dash or "not picked" in #95A1AE, no box). Keep exactly ONE dashed affordance per region: the YAML drop zone on the left, and the single "Not yet measured" tile in the rail. Do not repeat a dashed treatment four times in one column.
