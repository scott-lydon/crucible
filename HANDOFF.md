## 2. The direction (decided 2026-06-22)
Crucible targets the **Integrity** pillar of the CIA triad, *only* Integrity: catch
**silently-wrong model outputs** (reward hacking, specification gaming, hallucination). The loop:

- **RED** — LLM adversarial search finds inputs that make the model wrong (strategy catalog, white-box, hybrid).
- **VERIFY** — **4 independent oracles over a sealed spec** (held-out tests, metamorphic relations, differential cross-family, property-fuzz) + an LLM judge (½ vote) → verdict + audit trace.
- **HARDEN (blue)** — reads the catalog, proposes features / adversarial training samples / specialist ensemble, **retrains** the target, validates on a **held-out** set it never trained on.
- **MEASURE** — dashboard, SSE traces, the headline **white-box catch-rate**, exported corpus, SR-11-7 report, halt-certification.

Targets: fraud (LightGBM / Kaggle creditcard), code-agent (Sonnet producer), research-agent (stub).
Stack mandated by `plan.md`: **FastAPI orchestrator + Postgres (Supabase) + Modal sandbox + React/Vite dashboard + SSE**; models **Opus 4.8 / Sonnet 4.6**. 4 pillar owners; 15 user stories.

## 3. Relationship to the earlier branch `julian/devtool-mvp`
That branch is a **different product** — AI-agent *behavioral security* (prompt injection, secret
leakage, tool abuse, jailbreaks). `main` **explicitly scopes that OUT** ("What Crucible is not: a
secrets-exfiltration or prompt-injection scanner"). It is working, tested code, but solves the
*Confidentiality/behavioral* problem, not *Integrity*. Treat it as parked/standalone.

**Reusable from it (a head start, not the whole):**
- LLM **red-search engine** (search + strategy catalog + best-of-N/TAP + iterate-on-block) → `modules/red/` (`search.py`, `white_box.py`, `hybrid.py`, `catalog.py`).
- **Held-out vs validation methodology** ("never test on the attacks you fixed against") → Measure + blue held-out validation.
- **LLM client + cost meter** (OpenRouter/Anthropic), **mock-LLM mode** (`ScriptedLLM` ≈ US-15), report/export generation, loop-orchestration patterns.
- **Not reusable:** the 4 correctness oracles, sealed spec, Modal sandbox, retrain blue loop, Postgres, React dashboard, SR-11-7, halt-cert — all new.
